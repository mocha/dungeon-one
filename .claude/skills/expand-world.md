# Expand World

Generate multiple interconnected entries to flesh out a part of the world.

## Process

1. **Understand the scope**:
   - What area is being expanded? A settlement? A region? A faction's network?
   - Read all existing entries related to this area — locations, characters, factions, lore, creatures
   - Understand what already exists so new content fills gaps rather than duplicating

2. **Propose an expansion plan** before generating anything:
   - Based on what exists and what's thin, suggest what to create
   - Example: "Thornhaven has a location entry but only one NPC. I'd suggest adding: the innkeeper at The Pickaxe Inn, the mine foreman, 2-3 other notable residents, a couple of landmarks (the mine entrance, the market square), and maybe a local legend. Want me to generate all of these, or pick and choose?"
   - Let the DM approve, modify, or add to the plan
   - Offer a rough count: "That would be about 8 new entries — 4 characters, 3 locations, 1 lore piece."

3. **Generate entries in relationship order**:
   - Create locations first (they anchor everything else)
   - Then factions (they provide organizational context)
   - Then characters (they reference locations and factions)
   - Then creatures, items, and lore (they reference all of the above)
   - Each entry should reference the others — the whole batch should feel interconnected

4. **Maintain world consistency**:
   - Read existing entries before each new file to avoid contradictions
   - Match the naming conventions, tone, and level of detail of existing content
   - New characters should have relationships to existing ones where it makes sense
   - New factions should have opinions about existing factions

5. **Present a summary** when done:
   - List everything created with one-line descriptions
   - Highlight new connections to existing content
   - Flag any dangling references that might want future entries
   - Suggest what could be expanded next

6. **Update all llms.txt files** for directories that received new entries.

## Key Rules
- Always propose before generating. Don't surprise the DM with 20 new entries.
- Interconnection is the goal. Isolated entries are less useful than a web of related content.
- Generate at the level of detail that matches the existing world. If existing entries are sparse, keep new ones sparse. If they're rich, match that richness.
- Leave hooks and mysteries. Not everything needs to be explained. A mysterious locked door or an NPC who won't talk about their past creates future story potential.
- Every character needs a reason to be here and something they want. No filler NPCs.
